## 2. 프레이밍 라벨 정의 (6가지 유형)

| 유형 | 영문 | 핵심 기준 | 대표 표현 |
|------|------|-----------|-----------|
| 낙관적 | optimistic | 긍정적 전망, 좋은 성과 강조 | 성장, 호조, 반등, 최고치, 흑자 전환 |
| 비관적 | pessimistic | 부정적 현실, 하락/악화 보도 | 둔화, 하락, 감소, 위축, 부진, 침체 |
| 경보적 | alarmist | 극단적 위기감, 공포 조장 | 폭락, 붕괴, 공포, 패닉, 시한폭탄, 위기 |
| 방어적 | defensive | 부정적 상황에도 긍정적 결과 (반전 구조) | ~에도 불구하고, ~속에서도, 선방, 극복 |
| 비교적 | comparative | 두 대상의 명시적 비교가 핵심 | A vs B, 격차, 순위, 꼴찌, 추월 |
| 중립적 | neutral | 감정 없는 사실 전달 | 동결, 보합, 발표, 집계, (종합) |

### 판단 플로우차트 (우선순위)

```
1. 극단적 공포/위기감? → 경보적
2. "~에도" 반전 구조?  → 방어적
3. 비교가 기사의 핵심? → 비교적
4. 긍정 성과/전망?     → 낙관적
5. 부정 현실/하락?     → 비관적
6. 위 해당 없음        → 중립적
```

### 주요 판단 원칙
- **제목 우선**: 제목이 판단의 80%, 본문은 참고용
- **하나만 선택**: 복합적이어도 가장 강한 프레이밍 하나만
- **맥락 판단**: "역대 최대 적자"는 낙관이 아닌 비관 (수식어가 아니라 전체 맥락)
- **비교적 구분**: "전년 대비 3% 성장"은 단순 수치 보고 → 비교적 아님. 비교 자체가 주제인 경우만.

## 3. 라벨링 과정

### 3-1. 1차 수동 라벨링 (1,000건)
- **방법**: dataset.csv에서 event_type별 균등 샘플링 → 직접 제목을 읽고 판단
- **결과**: `data/labeled/labeled_1000.csv`

### 3-2. 2차 수동 라벨링 (2,000건)
- **방법**: 1차와 중복 없이 추가 2,000건 샘플링 → 직접 라벨링
- **결과**: `data/labeled/labeled_2000.csv`

### 3-3. 통합 (3,262건)
- 1차 + 2차 합산 후 중복 제거: **3,000건**
- defensive(57건), comparative(88건) 클래스 부족 → **262건 추가 라벨링**
  - 반전 구조 키워드("~에도", "극복", "선방" 등) 포함 기사에서 defensive 후보 추출
  - 비교 키워드("vs", "격차", "꼴찌" 등) 포함 기사에서 comparative 후보 추출
  - 262건 전수 직접 라벨링
- **최종 수동 라벨 데이터**: `data/labeled/labeled_3000.csv` (3,262건)

### 3-4. 모델 학습 및 자동 라벨링

#### 1차 시도: snunlp/KR-FinBert-SC (110M)
- 한국 금융 뉴스 감성분류 파인튜닝 모델
- **결과**: Macro F1 0.63, Accuracy 68%
- defensive(0.51), pessimistic(0.55) 성능 부족

#### 2차 시도: klue/roberta-large (335M)
- 범용 한국어 대형 모델
- 하이퍼파라미터: lr=1e-5, batch=8, epochs=15, max_length=128
- WeightedCrossEntropy로 클래스 불균형 보정
- **결과**: Macro F1 0.74, Accuracy 76%

| 클래스 | precision | recall | f1-score |
|--------|-----------|--------|----------|
| alarmist | 0.78 | 0.73 | 0.76 |
| comparative | 0.88 | 0.72 | 0.79 |
| defensive | 0.53 | 0.71 | 0.61 |
| neutral | 0.78 | 0.82 | 0.80 |
| optimistic | 0.87 | 0.66 | 0.75 |
| pessimistic | 0.66 | 0.76 | 0.71 |

#### 자동 라벨링
- 학습된 roberta-large 모델로 전체 13,886건 추론
- 저신뢰도 120건(confidence < 0.5) 전수 검수 → **62건 수정**
- **최종 결과**: `data/labeled/auto_labeled_full.csv`

## 4. 데이터 품질 이슈 및 처리

### 4-1. 크롤링 오류에 의한 content 중복

크롤링 과정에서 일부 매체의 본문이 정상적으로 수집되지 않은 문제를 발견함.

| 매체 | 건수 | content 중복률 | date 누락 | 원인 |
|------|------|---------------|-----------|------|
| 매일경제TV | 160건 | 98.1% | 100% | 영상 뉴스 페이지 → 사이드바/관련기사 목록이 본문으로 수집됨 |
| 서울경제TV | 87건 | 10.3% | 100% | 동일 원인 |
| 미주중앙일보 | 41건 | 2.4% | 100% | 페이지 구조 파싱 실패 |
| 기타 매체 | - | 0.1~1.3% | <0.1% | 정상 |

### 4-2. 처리 방법

```python
# 1. 크롤링 오류 매체의 content 제거 (title만 사용)
BAD_CONTENT_MEDIA = ["매일경제TV", "서울경제TV", "미주중앙일보"]

# 2. 동일 content가 5건 이상 공유된 경우도 제거
#    (매체와 무관하게 크롤링 오류로 판단)

# 3. 정상 기사: title [SEP] content[:500] 형태로 결합
# 4. 오류 기사: title만 사용
```

- 이 매체들의 **제목(title)은 정상**이므로 라벨링에는 문제 없음
- 학습 시 해당 기사는 title만으로 학습됨
